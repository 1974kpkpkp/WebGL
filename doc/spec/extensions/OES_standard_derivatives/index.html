Overview

This extension exposes the OES_standard_derivatives functionality to WebGL. There are no WebGL-specific behavioral changes. Consult the above extension for all documentation, issues, and new functions and enumerants.

When this extension is enabled:

  • The hint entry point accepts FRAGMENT_SHADER_DERIVATIVE_HINT_OES as a target.
  • The built-in functions dFdx(), dFdy(), and fwidth() are available in fragment shaders if the following directive is added to the shader: 
    #extension GL_OES_standard_derivatives : enable
  • The following macro definition is available: 
    #define GL_OES_standard_derivatives 1

IDL

module webgl {

interface OES_standard_derivatives {
    const GLenum FRAGMENT_SHADER_DERIVATIVE_HINT_OES = 0x8B8B;
};

};
