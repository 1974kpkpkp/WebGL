Overview

This extension exposes new tokens for querying the underlying graphics driver's VENDOR and RENDERER strings.

IDL

module webgl {

interface WEBGL_debug_gpu_info {

      const GLenum UNMASKED_VENDOR            = 0x6901; ??
      const GLenum UNMASKED_RENDERER          = 0x6902; ??

};

};

New Tokens

any getParameter(GLenum pname)
Two new enums UNMASKED_VENDOR and UNMASKED_RENDERER are accepted by pname parameter in getParameter().
pnamereturned type
UNMASKED_VENDORDOMString
UNMASKED_RENDERERDOMString

The following pname arguments return a string describing some aspect of the underlying graphics driver.
UNMASKED_VENDORReturn the VENDOR string of the underlying graphics driver.
UNMASKED_RENDERERReturn the RENDERER string of the underlying graphics driver.

Security Considerations

VENDOR and RENDERER strings from the underlying graphics driver could be used to identify a user's GPU, which in turn could be used to identify a unique user. Therefore, this extension has privacy implications and MUST NOT be available (see getSupportedExtensions()) to regular web content. The extension shall be available only to privileged code in the browser.
