Overview

This extension exposes new functions and tokens that expose the shader source passed down to the graphics driver, the unmasked VENDOR and RENDERER strings from the graphics driver. Because such information could be used to identify a user's GPU, which in turn could be used to identify an unique user, it is strongly recommended that this extension should be available only to privilidged applications, for example, Chrome extensions.

IDL

module webgl {

interface WEBGL_debug_shaders {

      const GLenum UNMASKED_VENDOR                    = 0x6901;
      const GLenum UNMASKED_RENDERER                  = 0x6902;

      DOMString getTranslatedShaderSource(WebGLShader shader);

};

};

New Procedures and Functions

DOMString getTranslatedShaderSource(WebGLShader shader)

WebGL uses the GLSL ES 2.0 spec on all platforms, and translates these shaders to the host platform's native language (HLSL, GLSL, and even GLSL ES). For debugging purpose, it is useful to be able to examine the shader after translation. If no source has been defined, compileShader has not been called, or the translation has failed for shader, an empty string is returned.

New Tokens

UNMASKED_VENDOR and UNMASKED_RENDERER are accepted by getParameter(), which returns the unmasked VENDOR and RENDERER strings from the underlying graphics driver.
